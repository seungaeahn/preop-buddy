# PreOp Buddy — 프로젝트 로드맵

> 마지막 업데이트: 2026-03-13

---

## 현재 상태

**Phase 0 (완료)** — 기획 및 저장소 구성

---

## Phase 1 — MVP (해커톤 당일)

**목표:** 수술명을 입력하면 AI가 정보를 생성하는 동작하는 프로토타입

### 마일스톤

| # | 작업 | 상태 |
|---|---|---|
| 1-1 | Vite + React + Tailwind 프로젝트 초기 설정 | ⬜ 대기 |
| 1-2 | Claude API 연동 + 스트리밍 처리 | ⬜ 대기 |
| 1-3 | 프롬프트 설계 (난이도 3단계, JSON 출력) | ⬜ 대기 |
| 1-4 | `SearchInput` 컴포넌트 (입력 + 자동완성) | ⬜ 대기 |
| 1-5 | `ResultCard` 컴포넌트 (5개 섹션 렌더링) | ⬜ 대기 |
| 1-6 | `DifficultySelector` 컴포넌트 | ⬜ 대기 |
| 1-7 | `Checklist` 컴포넌트 (인터랙티브 체크박스) | ⬜ 대기 |
| 1-8 | 면책 조항 고정 표시 | ⬜ 대기 |
| 1-9 | 반응형 UI (모바일 우선) | ⬜ 대기 |
| 1-10 | Vercel 배포 | ⬜ 대기 |

### 완료 기준
- [ ] 수술명 입력 → AI 정보 생성 → 화면 표시 동작
- [ ] 난이도 3단계 전환 동작
- [ ] 체크리스트 인터랙션 동작
- [ ] 모바일에서 정상 표시
- [ ] Vercel URL로 접근 가능

## Phase 2 — 품질 개선 (해커톤 이후 1~2주)

**목표:** 사용성 및 안정성 향상

| # | 작업 | 우선순위 |
|---|---|---|
| 2-1 | 에러 처리 강화 (API 실패, 타임아웃, 알 수 없는 수술명) | 높음 |
| 2-2 | 로딩 스켈레톤 UI | 중 |
| 2-3 | 수술명 자동완성 목록 확장 (50개 → 200개) | 중 |
| 2-4 | 결과 인쇄 / PDF 저장 기능 | 중 |
| 2-5 | 접근성 개선 (WCAG AA, 스크린 리더) | 높음 |
| 2-6 | Google Analytics 연동 (사용 완료율 측정) | 중 |
| 2-7 | 의료진 리뷰를 통한 정확도 검증 (10개 샘플) | 높음 |

---

## Phase 3 — 가설 검증 (해커톤 이후 1개월)

**목표:** H1·H2 가설 검증 및 데이터 수집

| # | 작업 | 우선순위 |
|---|---|---|
| 3-1 | 사용자 이해도 퀴즈 (5문항) 삽입 | 높음 |
| 3-2 | 수술 전 불안도 VAS 설문 삽입 | 높음 |
| 3-3 | 사용자 피드백 폼 ("도움이 되었나요?") | 중 |
| 3-4 | 지표 대시보드 구성 (이해도, 불안도, 완료율) | 중 |
| 3-5 | 파일럿 테스트: 실제 수술 환자 10명 대상 | 높음 |

---

## Phase 4 — 기능 확장 (3개월 이후)

**목표:** 실제 의료 현장 적용 가능 수준으로 발전

| # | 작업 | 우선순위 |
|---|---|---|
| 4-1 | 다국어 지원 (영어, 중국어 — 외국인 환자) | 중 |
| 4-2 | TTS 음성 읽기 (고령 사용자 접근성) | 중 |
| 4-3 | 병원별 커스텀 준비사항 추가 기능 | 높음 |
| 4-4 | 수술 정보 즐겨찾기 / 북마크 | 낮음 |
| 4-5 | 백엔드 API 서버 분리 (보안 강화) | 높음 |
| 4-6 | 의료진용 관리 대시보드 | 중 |

## 장기 비전

```
2026 Q1  ──▶  MVP 해커톤 출품
2026 Q2  ──▶  가설 검증 완료, 실제 환자 파일럿
2026 Q3  ──▶  병원 1~2곳과 파트너십 논의
2026 Q4  ──▶  실제 병원 환자에게 서비스 제공
```

**궁극적 목표:** 수술 예정 환자가 병원에서 태블릿으로 PreOp Buddy를 사용하고, 담당 의료진의 설명 시간을 30% 단축한다.
