# PRD — PreOp Buddy

> 제품 요구사항 문서 (Product Requirements Document)
> 버전: 1.0 | 작성일: 2026-03-13 | DevDay 해커톤 출품작

---

## 1. 개요

### 1.1 배경 및 문제 정의

한국의 수술 환자는 수술 동의서에 서명하지만, 내용을 제대로 이해하지 못하는 경우가 대부분이다.

**현재 문제:**
- 수술 동의서는 법률·의학 용어로 작성되어 일반인이 이해하기 어렵다
- 의료진은 하루 수십 명의 환자에게 동일한 설명을 반복할 시간이 없다
- 환자가 직접 인터넷에서 검색하면 부정확하거나 공포를 유발하는 정보에 노출된다
- 이로 인해 수술 전 불안이 높아지고, 불필요한 질문이 의료진에게 집중된다

### 1.2 솔루션

**PreOp Buddy**는 수술명을 입력하면 AI가 환자 눈높이에 맞춘 수술 정보를 자동 생성하는 웹 애플리케이션이다.

- 의학 용어를 환자가 이해할 수 있는 언어로 변환
- 수술 준비사항을 인터랙티브 체크리스트로 제공
- 설명 난이도를 사용자가 직접 조절 가능

### 1.3 타겟 사용자

| 구분 | 설명 |
|---|---|
| 1차 사용자 | 수술 예정 환자 (40~70대, 디지털 리터러시 낮음~중간) |
| 2차 사용자 | 환자 보호자 (환자 대신 정보 탐색) |
| 비사용자 | 의료진 (정보 생성의 주체가 아님) |

## 2. 목표 및 성공 지표

### 2.1 제품 목표

1. 수술 전 환자의 **정보 이해도 향상**
2. 수술 전 환자의 **불안감 감소**
3. 의료진의 **반복 설명 업무 경감** (간접 효과)

### 2.2 가설

| ID | 가설 | 검증 방법 | 검증 시점 |
|---|---|---|---|
| H1 | PreOp Buddy 사용 시, 환자의 수술 이해도가 기존 동의서 대비 50% 이상 향상된다 | 수술별 5문항 퀴즈 사전·사후 비교 (10명 파일럿) | D+30 |
| H2 | PreOp Buddy 사용 환자의 수술 전 불안 점수(VAS)가 20% 이상 감소한다 | VAS 0~10 슬라이더 사전·사후 비교 (10명 파일럿) | D+30 |
| H3 | AI 생성 정보의 의학적 정확도가 90% 이상이다 | 의료진 리뷰 체크리스트 (10개 수술 샘플, 8항목 평가표) | D+0 (자체), D+14 (전문가) |

### 2.3 성공 지표 (KPI)

| 지표 | 측정 방법 | 측정 도구 | 목표 |
|---|---|---|---|
| 수술 이해도 향상율 | 앱 내 5문항 퀴즈 전/후 점수 비교 | 앱 내 퀴즈 모듈 (D+7 구현 예정) | +50% |
| 수술 전 불안도 감소 | 앱 내 VAS 슬라이더 전/후 비교 | 앱 내 VAS 모듈 (D+7 구현 예정) | -20% |
| 정보 정확도 | 의료진 리뷰 체크리스트 (수술당 10항목) | [VALIDATION.md](./VALIDATION.md) 2-1 참고 | 90% 이상 |
| 세션 완료율 | 마지막 섹션(용어사전) 도달 비율 | Google Analytics 이벤트 (D+3 구현 예정) | 70% 이상 |
| 체크리스트 활용율 | 체크박스 1개 이상 클릭한 사용자 비율 | Google Analytics 이벤트 | 60% 이상 |
| SUS 사용성 점수 | 10문항 SUS 설문 | Google Forms | 68점 이상 |
| NPS | "주변에 추천하겠다" 0~10 | 세션 종료 팝업 (D+14 구현 예정) | 50 이상 |

## 3. 기능 요구사항

### 3.1 핵심 기능 (Must Have)

#### F-01. 수술명 입력 및 자동완성
- 사용자가 수술명을 한국어 또는 영문으로 입력할 수 있다
- 자주 사용되는 수술명 목록을 기반으로 자동완성을 제공한다
- 입력값이 비어있을 때 검색 버튼을 비활성화한다

#### F-02. AI 수술 정보 생성
- 수술명을 입력하면 Claude API를 호출하여 5개 섹션의 구조화된 정보를 생성한다
  - **수술 개요:** 어떤 수술인지 쉬운 비유를 활용하여 설명
  - **수술 과정:** 마취부터 회복실까지 단계별 순서 안내
  - **수술 전 준비사항:** 금식, 약물, 준비물 등 체크리스트
  - **회복 과정:** 입원 기간, 일상 복귀 시점, 주의사항
  - **자주 묻는 질문(FAQ):** 환자가 가장 많이 묻는 5가지
- 스트리밍 방식으로 응답하여 사용자 체감 대기시간을 줄인다
- 알 수 없는 수술명 입력 시 친절한 오류 메시지를 표시한다

#### F-03. 설명 난이도 조절
- 3단계 난이도를 제공한다:
  - **쉽게:** 초등학생도 이해할 수 있는 일상 언어
  - **자세하게:** 일반 성인 수준의 상세한 설명
  - **의학 용어 포함:** 의학 용어를 병기하여 설명
- 기본값은 "쉽게"로 설정한다
- 난이도 변경 시 즉시 재생성한다

#### F-04. 수술 전 체크리스트
- 준비사항 항목을 인터랙티브 체크박스로 표시한다
- 각 항목을 클릭하여 완료 표시(체크)할 수 있다
- 체크 상태는 세션 내에서 유지된다
- 전체 항목 수와 완료 항목 수를 표시한다

#### F-05. 면책 조항 표시
- 모든 생성 결과 하단에 면책 조항을 표시한다
- 면책 조항: "이 정보는 일반적인 참고 자료이며 의학적 조언을 대체하지 않습니다. 반드시 담당 의료진과 상담하세요."
- 면책 조항은 숨기거나 삭제할 수 없다

### 3.2 부가 기능 (Nice to Have, 해커톤 이후)

| 기능 | 설명 | 우선순위 |
|---|---|---|
| 결과 인쇄/PDF 저장 | 생성된 정보를 인쇄하거나 PDF로 저장 | 중 |
| 즐겨찾기 | 자주 조회하는 수술 정보 저장 | 하 |
| 다국어 지원 | 영어, 중국어 등 외국인 환자 지원 | 중 |
| 음성 읽기 | TTS로 정보를 음성으로 읽어주기 | 하 |
| 병원 정보 연동 | 수술 병원의 특이 준비사항 반영 | 하 |

### 3.3 명시적 제외 기능

- 특정 의사/병원 추천
- 진단 또는 처방 기능
- 환자 개인정보 입력 (이름, 주민번호, 연락처 등)

## 4. 비기능 요구사항

### 4.1 접근성
- 최소 폰트 크기: 16px
- 터치 영역: 버튼 최소 44px × 44px (모바일)
- 색상 대비: WCAG AA 기준 충족 (대비율 4.5:1 이상)
- 스크린 리더 호환 (시맨틱 HTML 사용)

### 4.2 성능
- 첫 바이트 응답 시간: 3초 이내
- AI 생성 첫 텍스트 노출: 2초 이내 (스트리밍)
- Lighthouse 성능 점수: 80 이상

### 4.3 반응형
- 모바일 (375px~) / 태블릿 (768px~) / 데스크톱 (1280px~) 지원
- 모바일 우선(Mobile First) 설계

### 4.4 보안
- API 키는 환경변수로만 관리 (코드 하드코딩 금지)
- 사용자 입력값 XSS 방지 처리
- HTTPS 통신만 허용 (Vercel 기본 제공)

## 5. 기술 스택

| 영역 | 기술 | 버전 | 선택 이유 |
|---|---|---|---|
| 프론트엔드 | React | 18 | 컴포넌트 재사용, 빠른 개발 |
| 빌드 도구 | Vite | 6 | 빠른 HMR, 간단한 설정 |
| 스타일링 | Tailwind CSS | 4 | 빠른 UI 구현, rem 기반 폰트 스케일링 |
| AI | Groq API (llama-3.3-70b-versatile) | - | 빠른 응답 속도, JSON mode, 무료 티어 |
| 배포 | Vercel | - | Git push → 자동 배포, 무료 |

### 아키텍처 다이어그램

```
[사용자 브라우저]
       │
       ▼
[React SPA on Vercel]
  ├── SearchInput        ← 수술명 입력 + 토큰 기반 자동완성
  ├── ResultCard         ← 섹션 페이지네이션 컨테이너
  │   ├── OverviewIllustration  ← 수술별 SVG 일러스트 (14종)
  │   ├── MedicalText           ← 인라인 의학 용어 툴팁
  │   ├── DdayPlanner           ← D-day 카운트다운 + 타임라인
  │   └── Checklist             ← 인터랙티브 체크리스트
  └── Buddy              ← 캐릭터 + 말풍선 내레이터
       │
       ▼
[groqApi.js]  ← JSON mode, temperature 0.3, max_tokens 4000
  └── surgeryPrompt.js  ← 한국어 전용 프롬프트 + 스키마 강제
       │
       ▼
[Groq API — llama-3.3-70b-versatile]
  └── JSON 응답 (surgeryName, overview, process, preparationTimeline,
                  preparation, recovery, faq, glossary, disclaimer)
```

백엔드 서버 없이 프론트엔드에서 Groq API를 직접 호출하는 단순한 구조. 해커톤 기간 내 빠른 구현에 최적화.

## 6. 사용자 흐름 (User Flow)

```
1. 랜딩 페이지 접속
   └── 서비스 소개 + 검색창 표시

2. 수술명 입력
   ├── 직접 타이핑 또는 자동완성 선택
   └── 난이도 선택 (기본: 쉽게)

3. 검색 버튼 클릭
   └── 로딩 상태 표시 (스트리밍 시작)

4. 결과 표시 (스트리밍으로 순차 렌더링)
   ├── 수술 개요
   ├── 수술 과정
   ├── 수술 전 준비사항 (체크리스트)
   ├── 회복 과정
   ├── FAQ
   └── 면책 조항

5. 인터랙션
   ├── 체크리스트 항목 체크
   ├── 난이도 변경 → 재생성
   └── 새 수술명 검색
```

## 8. API 설계

### 8.1 Claude API 요청 포맷

```javascript
{
  model: "claude-sonnet-4-20250514",
  max_tokens: 2000,
  stream: true,
  system: "당신은 환자 교육 전문가입니다. ...[난이도별 지침]...",
  messages: [
    {
      role: "user",
      content: `수술명: ${surgeryName}\n다음 JSON 형식으로 응답해주세요: ...`
    }
  ]
}
```

### 8.2 응답 JSON 스키마

```json
{
  "surgeryName": "string",
  "overview": "string",
  "process": [
    { "step": "number", "title": "string", "description": "string" }
  ],
  "preparation": [
    { "item": "string", "category": "string (식이|약물|준비물|기타)" }
  ],
  "recovery": {
    "hospitalDays": "string",
    "returnToDaily": "string",
    "precautions": ["string"]
  },
  "faq": [
    { "question": "string", "answer": "string" }
  ],
  "disclaimer": "string"
}
```
