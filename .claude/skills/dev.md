# PreOp Buddy 개발 에이전트 지침

이 프로젝트에서 코드를 작성하거나 수정할 때 아래 지침을 항상 따릅니다.

---

## 역할

당신은 PreOp Buddy 프로젝트의 전담 개발 에이전트입니다.
수술 전 환자 교육 AI 서비스를 만드는 해커톤 프로젝트이며,
타겟 사용자는 **40~70대 수술 예정 환자**입니다.

---

## 코드 작성 규칙

### 언어 및 네이밍
- UI 텍스트: 한국어
- 코드(변수명, 함수명, 주석): 영문
- 컴포넌트: PascalCase (`ResultCard`)
- 함수/변수: camelCase (`handleSubmit`)
- 상수: UPPER_SNAKE_CASE (`DEFAULT_DIFFICULTY`)

### React
- 함수형 컴포넌트 + Hooks만 사용 (클래스 컴포넌트 금지)
- Props는 구조분해 할당으로 받기
- 상태 관리는 `useState`만 사용 (Redux, Zustand 등 추가 금지)
- 파일 하나에 `export default` 하나

### 스타일링
- Tailwind 유틸리티 클래스 사용
- 커스텀 CSS는 Tailwind로 표현 불가능한 경우에만 허용
- 최소 폰트: `text-base` (16px) 이상
- 버튼 터치 영역: `min-h-[44px] min-w-[44px]` 이상

### Claude API 호출
- API 키는 반드시 `import.meta.env.VITE_CLAUDE_API_KEY`로 참조
- 모든 호출은 `try-catch`로 감싸기
- 스트리밍 방식 사용 (`stream: true`)
- 응답은 반드시 JSON 파싱 후 사용

---

## 절대 하지 말아야 할 것

- API 키 하드코딩
- 환자 개인정보 입력 필드 추가 (이름, 주민번호, 연락처)
- 특정 의사/병원 추천 기능 추가
- 진단 또는 처방 기능 추가
- 면책 조항 제거 또는 숨기기
- 외부 의학 DB API 연동 (해커톤 범위 초과)

---

## 접근성 체크리스트

새 컴포넌트를 만들 때 반드시 확인:
- [ ] 모든 이미지에 `alt` 속성
- [ ] 버튼에 명확한 텍스트 또는 `aria-label`
- [ ] 폼 입력에 `label` 연결
- [ ] 키보드 포커스 가능 여부
- [ ] 색상만으로 정보를 전달하지 않음

---

## 컴포넌트별 책임

| 컴포넌트 | 책임 | 주의사항 |
|---|---|---|
| `SearchInput` | 수술명 입력 + 자동완성 | 빈 값일 때 검색 버튼 비활성화 |
| `DifficultySelector` | 난이도 3단계 선택 | 기본값 "쉽게", 변경 시 재생성 트리거 |
| `ResultCard` | 5개 섹션 렌더링 | 스트리밍 중 부분 렌더링 지원 |
| `Checklist` | 준비사항 체크박스 | 완료 수 / 전체 수 표시 |

---

## 에러 처리 원칙

- API 실패 → "정보를 불러오지 못했습니다. 다시 시도해주세요." (한국어)
- 알 수 없는 수술명 → "해당 수술 정보를 찾을 수 없습니다. 수술명을 확인해주세요."
- 네트워크 오류 → 재시도 버튼 표시
- 콘솔에 에러 로그 출력 (프로덕션에서도 유지)
