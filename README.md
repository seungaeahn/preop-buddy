# 🏥 PreOp Buddy — 수술 전 환자 교육 AI 어시스턴트

> "수술이 무섭지 않도록, AI가 미리 알려드립니다"

수술명을 입력하면 AI가 **환자 눈높이에 맞춘 수술 설명, 준비사항, 회복 과정, FAQ**를 자동으로 생성해주는 웹 애플리케이션입니다.

## 🔗 배포 URL

> **https://preop-buddy.vercel.app**

---

## 🤔 왜 만들었나요?

한국에서 수술을 앞둔 환자 대부분은 **수술 동의서에 서명하지만, 내용을 제대로 이해하지 못합니다.**

| 문제 | 현실 |
|---|---|
| 동의서는 법률·의학 용어로 가득 | 일반인이 이해하기 어려움 |
| 의료진 시간 부족 | 하루 수십 명 반복 설명 불가 |
| 환자 직접 인터넷 검색 | 부정확하거나 공포 유발 정보 노출 |
| 결과 | 수술 전 불안 증가, 불필요한 질문 집중 |

**PreOp Buddy는 이 간극을 메웁니다.** AI가 신뢰할 수 있는 수준의 수술 정보를 환자가 이해할 수 있는 언어로 즉시 변환해줍니다.

### 기존 서비스와의 차별점

| 구분 | 포털 의학 정보 | 병원 안내문 | PreOp Buddy |
|---|---|---|---|
| 개인화 | ❌ 일반 정보 | ❌ 병원 표준 | ✅ 수술별 맞춤 생성 |
| 언어 수준 | ❌ 의학 용어 다수 | ❌ 어려운 표현 | ✅ 환자 눈높이 + 용어 툴팁 |
| 인터랙션 | ❌ 읽기만 가능 | ❌ 종이 문서 | ✅ 체크리스트, D-day, 퀴즈 |
| 접근성 | ❌ 고령자 배려 없음 | ❌ 작은 글씨 | ✅ 글자 크기 3단계 조절 |
| 불안 완화 | ❌ 무서운 사진 | ❌ 딱딱한 설명 | ✅ VAS 불안도 측정 + 친근한 캐릭터 |
| 검증 도구 | ❌ 없음 | ❌ 없음 | ✅ 앱 내 이해도 퀴즈 + VAS |

### AI 기술 차별화

PreOp Buddy의 핵심은 **프롬프트 엔지니어링**에 있습니다.

| 기술 요소 | 접근 방식 | 효과 |
|---|---|---|
| **JSON Schema 강제** | 시스템 프롬프트에 완전한 JSON 스키마 명시 | 파싱 오류 0%, 구조 안정성 확보 |
| **한국어 전용 규칙** | 나쁜 예/좋은 예 명시 + JS 후처리(Unicode 필터) | 한자·영문 혼입 차단 이중 방어 |
| **temperature 0.3** | 창의성 억제, 의학 정보 일관성 우선 | 매 요청 동일한 구조·품질 유지 |
| **퀴즈 자동 생성** | AI가 본문 기반 3지선다 문항 자동 설계 | 별도 문항 DB 없이 수술별 검증 가능 |
| **모델 선택 (llama-3.3-70b)** | Groq 인프라로 2-3초 응답 | GPT-4 대비 10배 빠른 속도, 무료 |

## ✨ 핵심 기능

### 1. AI 수술 정보 생성
수술명 입력 → **7개 섹션**의 구조화된 정보 즉시 생성:

| 섹션 | 내용 |
|---|---|
| 📋 수술 개요 | 어떤 수술인지 쉬운 언어로 설명 + 수술별 일러스트 |
| 🔬 수술 과정 | 마취→수술→회복실까지 단계별 타임라인 |
| 📅 D-day 플래너 | 수술 날짜 입력 → D-14/D-7/D-3/D-1/D-0 준비 일정 |
| ✅ 수술 전 준비사항 | 인터랙티브 체크리스트 (식이/약물/준비물 카테고리) |
| 🌿 회복 과정 | 입원 기간, 일상 복귀 시점, 주의사항 |
| 💬 자주 묻는 질문 | 수술별 FAQ 아코디언 |
| 📖 용어 사전 | 어려운 의학 용어 인라인 툴팁 |

### 2. 섹션 페이지네이션 네비게이션
스크롤 대신 **이전/다음 버튼**으로 섹션을 한 장씩 넘기며 확인. 고령 사용자도 쉽게 탐색 가능.

### 3. D-day 준비 플래너
수술 예정일을 입력하면 **남은 일수(D-N)** 를 실시간 표시하고, AI가 생성한 날짜별 준비 체크리스트를 제공. 이미 지난 단계는 흐리게 표시.

### 4. 의학 용어 툴팁
생성된 텍스트에서 어려운 의학 용어를 자동 감지해 **터치/클릭 시 쉬운 설명 팝업**.

### 5. 글자 크기 조절
기본 / 크게 / 매우 크게 3단계. 브라우저 재시작 후에도 유지 (localStorage).

### 6. 인쇄 / PDF 저장
결과를 그대로 인쇄하거나 PDF로 저장. 배경색, 체크리스트 포함.

## 🛠️ 기술 스택

| 영역 | 기술 | 선택 이유 |
|---|---|---|
| 프론트엔드 | React 18 + Vite | 컴포넌트 재사용, 빠른 HMR |
| 스타일링 | Tailwind CSS v4 | 빠른 UI 구현, rem 기반 폰트 스케일링 |
| AI | Groq API (llama-3.3-70b-versatile) | 빠른 응답 속도, JSON mode 지원, 무료 티어 |
| 배포 | Vercel | Git push → 자동 배포 |
| 상태관리 | React useState / localStorage | 별도 백엔드 불필요 |

### 아키텍처

```
[사용자 브라우저]
       │
       ▼
[React SPA on Vercel]
  ├── SearchInput      ← 수술명 입력 + 자동완성
  ├── ResultCard       ← 섹션 페이지네이션 컨테이너
  │   ├── OverviewIllustration  ← SVG 일러스트
  │   ├── MedicalText           ← 인라인 용어 툴팁
  │   ├── DdayPlanner           ← D-day 타임라인
  │   └── Checklist             ← 인터랙티브 체크리스트
  └── Buddy            ← 캐릭터 + 말풍선 내레이터
       │
       ▼
[groqApi.js]  ← JSON mode, temperature 0.3
  └── surgeryPrompt.js ← 한국어 전용 프롬프트
       │
       ▼
[Groq API — llama-3.3-70b-versatile]
```

백엔드 서버 없이 프론트엔드에서 Groq API를 직접 호출하는 단순한 구조.

## 🚀 로컬 실행 방법

```bash
# 1. 저장소 클론
git clone https://github.com/seungaeahn/preop-buddy.git
cd preop-buddy

# 2. 의존성 설치
npm install

# 3. 환경 변수 설정
cp .env.example .env
# .env 파일에 VITE_GROQ_API_KEY=your_api_key 입력

# 4. 개발 서버 실행
npm run dev
```

`http://localhost:5173`에서 확인할 수 있습니다.

## 📊 검증 계획

### 핵심 가설

| ID | 가설 | 목표 |
|---|---|---|
| H1 | PreOp Buddy 사용 시 수술 이해도 향상 | 5문항 퀴즈 기준 +50% |
| H2 | 수술 전 불안도 감소 | VAS 0~10 기준 -20% |
| H3 | AI 생성 정보의 의학적 정확도 | 의료진 리뷰 90% 이상 |

### 해커톤 당일 즉시 검증
